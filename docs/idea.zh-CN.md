# PortBoard 项目构想

> 自动发现并管理电脑上所有本地开发服务的终端控制台。

## 要解决的问题

开发者经常同时运行前端、后端、数据库和 Docker 容器，却缺少一个统一入口来回答：

- 某个端口被哪个进程占用？
- 服务属于哪个 Git 项目？
- 服务是否健康，访问地址是什么？
- 哪些旧服务忘记关闭了？
- 手机如何访问电脑上的开发服务？

## MVP

- 扫描本机监听端口。
- 显示 PID、进程、工作目录、Git 仓库和启动命令。
- 识别 HTTP 服务并执行健康检查。
- 显示 Docker 端口映射。
- 支持复制 URL、打开浏览器和结束进程。
- 提供默认手动刷新、可选周期自动刷新的终端界面。
- 首先支持 macOS 和 Linux。

## 后续方向

- 生成局域网访问地址和二维码。
- 分配 `project.localhost` 固定域名。
- 保存常用服务和项目分组。
- 端口冲突提醒。
- 可选 Web UI 和系统托盘入口。

## 初始技术方案

- Python 3.11+
- Textual / Rich
- psutil
- HTTPX
- Docker CLI（可选集成）
- pytest

## 产品原则

- 无需账号、云服务或 API Key。
- 无需提前配置项目即可看到有用结果。
- 危险操作必须明确确认。
- README 中展示的功能必须已经可用。
- 优先做好一个核心工作流，再扩展平台和功能。
